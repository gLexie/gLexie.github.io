js立即执行函数：(function(){})()

发表于 2018-09-11

一、引申概念

要理解立即执行函数(function(){})()，先了解些函数的基本概念（函数声明、函数表达式、匿名函数）。

1、函数声明

  • 使用function声明函数，并指定函数名

2、函数表达式

  • 使用function声明函数，但未指定函数名，将匿名函数赋予一个变量

3、匿名函数

  • 使用function关键字声明函数，但未指定函数名。匿名函数属于函数表达式，匿名函数有很多作用，赋予一个变量则创建函数，赋予一个事件则成为事件处理程序或创建闭包等等

4、函数声明与函数表达式的不同

  • 函数声明可在当前作用域下提前调用执行，函数表达式需等执行到该函数后，方可执行，不可提前调用
  • 函数表达式可直接在函数后加括号调用

js方法：call() 和 apply()

发表于 2018-09-06

一、定义

ECMAScript规范为所有函数都包含两个方法（这两个方法非继承而来），call 和 apply 。这两个函数都是在特定的作用域中调用函数,能改变函数的作用域，实际上是改变函数体内 this 的值

1、call

  • 描述：用来代替另一个对象调用一个方法fn，可将一个函数的对象上下文从初始的上下文改变为由 thisObj 指定的新对象
  • 格式：call(thisObj, arg1, arg2…)
  • 参数：thisObj为更改后的函数调用上下文，如果没有提供 thisObj 参数，那么 Global 对象被用作 thisObj. arg1…为作用域下传递给函数的参数名，非必填

2、apply

  • 描述：功能同call
  • 格式：apply(thisObj, [argArray])
  • 参数：thisObj同call. 如果 argArray 不是一个有效的数组或者不是 arguments 对象，那么将导致一个 TypeError。如果没有提供 argArray 和 thisObj 任何一个参数，那么 Global 对象将被用作 thisObj， 并且无法被传递任何参数

二、区别

相同点:

  • 方法的含义是一样的，即方法功能是一样的
  • 第一个参数的作用是一样的

不同点：两者传入的列表形式不一样

  • call 可以传入多个参数，从第二个参数开始依次传入
  • apply 只能传入两个参数，所以其第二个参数往往是作为数组形式传入

三、高级用法，实现继承

四、总结

call 和 apply 最大的好处:方便我们解耦，对象不需要和方法有任何的耦合性，能使我们写出更好的面相对象程序。
大家如果看一些 js 框架底层的话会看到好多地方都有大量用到。

js方法：hasOwnProperty与in、isPrototypeOf与instanceof

发表于 2018-09-05

一、检查属性是否在对象中

1、hasOwnProperty

  • 描述：检查一个对象是否有属性，此方法无法检查该对象的原型链中是否具有该属性（该属性必须是对象本身的一个成员）
  • 格式：obj.hasOwnProperty(prop)
  • 参数：prop 要检测的属性 字符串 名称或者 Symbol
  • 返回值：用来判断某个对象是否含有指定的属性的 Boolean
  • 注意：和 in 运算符不同，该方法会忽略掉那些从原型链上继承到的属性

扩展：hasOwnProperty最好结合call使用

  • JavaScript 并没有保护 hasOwnProperty 属性名，因此某个对象是有可能存在使用这个属性名的属性，使用外部的 hasOwnProperty 获得正确的结果是需要的
  • 使用call在特定的作用域中调用函数,能改变函数的作用域，实际上是改变函数体内 this 的值

2、in

  • 描述：如果指定的属性在指定的对象或其原型链中，则in 运算符返回true
  • 格式：prop in object
  • 参数：prop 一个字符串类型或者 symbol 类型的属性名或者数组索引（非symbol类型将会强制转为字符串）. objectName 检查它（或其原型链）是否包含具有指定名称的属性的对象
  • 注意：in右操作数必须是一个对象值。例如，你可以指定使用String构造函数创建的字符串，但不能指定字符串文字

3、hasOwnProperty、in 对比

hasOwnProperty 只检查实例
in 检查实例和原型链

二、检查对象是否在原型链上

1、isPrototypeOf

  • 描述：检查一个对象（prototypeObj）是否存在于另一个对象（object）的原型链上
  • 格式：prototypeObj.isPrototypeOf(object)
  • 参数： object 在该对象的原型链上搜寻
  • 返回值： Boolean，表示调用对象是否在另一个对象的原型链上

2、instanceof

  • 描述：检测 constructor.prototype 是否存在于参数 object 的原型链上
  • 格式：object instanceof constructor
  • 参数: object 要检测的对象. constructor 某个构造函数

注意：

  • 如果表达式 obj instanceof Foo 返回true，则并不意味着该表达式会永远返回true，因为Foo.prototype属性的值有可能会改变，改变之后的值很有可能不存在于obj的原型链上，这时原表达式的值就会成为false。
  • 另外一种情况下，原表达式的值也会改变，就是改变对象obj的原型链的情况，虽然在目前的ES规范中，我们只能读取对象的原型而不能改变它，但借助于非标准的__proto__魔法属性，是可以实现的。比如执行obj.__proto__ = {}之后，obj instanceof Foo就会返回false了。

3、isPrototypeOf、instanceof 对比

A.isPrototypeOf(B) 判断的是A对象是否存在于B对象的原型链之中
A instanceof B 判断的是B.prototype是否存在于A的原型链之中
结论：如果 A.isPrototypeOf(B) 返回true，则 B instanceof A 一定返回true

vue-cli+webpack项目，手机无法访问本地服务

发表于 2018-08-10

需求

在pc端启动服务（使用npm run dev），用手机访问查看页面。

问题

使用npm run dev启动服务后，pc端可以通过 http://loaclhost:8888 来访问页面（其实也无法使用本机ip:port的地址去访问页面），手机与pc连接同一无线网络，也关闭了pc端防火墙，pc端也能ping通手机的ip，手机端却始终无法通过ip:port的地址去访问页面，会显示无法访问的字样。

解决

文件package.json中，在dev命令的最后加上–host 0.0.0.0：

“scripts”: {
“dev”: “webpack-dev-server –inline –progress –config build/webpack.dev.conf.js –host 0.0.0.0”
}

重新使用npm run dev启动服务即可用手机访问本地服务提供的页面了。
